# Contributing to Nylo

Thank you for your interest in contributing to Nylo! This document provides guidelines for contributing to the project.

## Contributor License Agreement

By submitting a pull request, you agree that:

1. Your contributions will be licensed under the [MIT License](LICENSE)
2. You have the right to submit the contribution
3. You grant the project maintainers a perpetual, worldwide, non-exclusive, royalty-free license to use, reproduce, modify, and distribute your contributions

## How to Contribute

### Reporting Issues

- Use [GitHub Issues](https://github.com/tejasgit/nylo/issues) to report bugs
- Include browser version, OS, and steps to reproduce
- Check existing issues before creating a new one

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Ensure all tests pass
5. Submit a pull request with a clear description

### Development Setup

```bash
git clone https://github.com/tejasgit/nylo.git
cd nylo
npm install
```

### Code Style

- Use consistent indentation (2 spaces)
- Add JSDoc comments for public APIs
- Follow existing naming conventions
- Keep functions focused and small
- Add input validation for all user-facing functions

### What We Accept

- Bug fixes
- Performance improvements
- Documentation improvements
- New tracking event types
- Test coverage improvements
- Security improvements

### What Requires Discussion First

Please open an issue to discuss before submitting PRs for:

- New cross-domain features (these may be covered by the commercial license)
- Breaking API changes
- Major architectural changes
- New dependencies

## Code of Conduct

- Be respectful and constructive
- Focus on the technical merits of contributions
- Welcome newcomers and help them get started

## Questions?

Open a [GitHub Discussion](https://github.com/tejasgit/nylo/discussions) for questions about contributing.
